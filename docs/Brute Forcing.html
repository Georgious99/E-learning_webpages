"Brute Forcing Login Page Password"

In this webpage, you are able to see a simple example of how hackers are using Social Engineering Data in order to try and guess the password to the corresponding User ID. In order to demonstrate this I have created a simple game representing this. On the right-hand side of the page, you will see a picture with some data that will help you guess the password of each person. Firstly you will need to enter the “User ID” in the text box on the left-hand side. Then by using the data provided on the right-hand side try and guess a password that may be the correct one and press the “Brute Force” button. The algorithm will try and guess the correct password by using all different combinations. If your first guess input is very close to the actual password the algorithm will not take a lot of time guessing the password, if it’s not it will take a little bit more time to find the correct password. You can use multiple inputs until the correct one is found and once found it will redirect you to the user’s webpage highlighting the correct password in green. As you go down the Social Engineering Data the more difficult it gets to guess the correct password.
